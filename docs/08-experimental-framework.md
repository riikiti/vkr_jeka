# 8. Экспериментальная часть

## 8.1 Цели экспериментов

1. Подтвердить работоспособность комбинированного метода.
2. Сравнить три подхода: чистый дифференциальный, чистый SAT, комбинированный.
3. Оценить масштабируемость по числу раундов.
4. Определить оптимальные параметры (порог вероятности, таймаут, выбор решателя).

---

## 8.2 Эксперимент 1: Валидация дифференциальных характеристик

**Цель:** Проверить корректность генерации дифференциальных характеристик и оценки вероятностей.

**Методика:**
1. Сгенерировать N характеристик для заданного числа раундов.
2. Для каждой характеристики произвести случайную выборку 2^k пар сообщений, удовлетворяющих входной разности.
3. Подсчитать долю пар, удовлетворяющих всей характеристике.
4. Сравнить экспериментальную вероятность с теоретической.

**Параметры:**

| Параметр | Значения |
|----------|---------|
| Хэш-функция | SHA-256, SHA-1 |
| Число раундов | 8, 12, 16 |
| Размер выборки (k) | 20, 24, 28 |
| Число характеристик (N) | 10–50 |

**Ожидаемый результат:**
- Экспериментальная вероятность близка к теоретической (±10%).
- Характеристики с малым весом Хэмминга имеют более высокую вероятность.

**Конфигурация:**
```yaml
experiment:
  name: "differential_validation"
  hash_function: "sha256"
  rounds: [8, 12, 16]
  sample_size_log2: [20, 24, 28]
  num_characteristics: 50
  probability_threshold: -40.0  # log2
  output_dir: "experiments/results/exp1"
  seed: 42
```

## 8.3 Эксперимент 2: Чистый SAT-поиск (без дифференциального направления)

**Цель:** Измерить базовую производительность SAT-решателя при поиске коллизий без дифференциальной информации.

**Методика:**
1. Закодировать задачу поиска коллизий (две копии хэш-функции, H(M) = H(M'), M ≠ M').
2. Запустить SAT-решатель с таймаутом.
3. Измерить время, число конфликтов, успешность.

**Параметры:**

| Параметр | Значения |
|----------|---------|
| Хэш-функция | SHA-256, SHA-1 |
| Число раундов | 8, 10, 12, 14, 16 |
| Решатели | CryptoMiniSat, MiniSAT, Glucose |
| Таймаут | 600с, 1800с, 3600с |

**Ожидаемый результат:**
- Для малого числа раундов (≤10) — SAT за приемлемое время.
- С ростом раундов — экспоненциальный рост времени.
- CryptoMiniSat быстрее на криптографических задачах благодаря Gauss elimination.

---

## 8.4 Эксперимент 3: Комбинированный метод (Differential + SAT)

**Цель:** Оценить эффективность комбинированного метода и сравнить с экспериментами 1–2.

**Методика:**
1. Сгенерировать топ-K дифференциальных характеристик.
2. Для каждой характеристики закодировать SAT-задачу с дифференциальными ограничениями.
3. Запустить SAT-решатель.
4. Измерить суммарное время (генерация + кодирование + решение).

**Параметры:**

| Параметр | Значения |
|----------|---------|
| Хэш-функция | SHA-256, SHA-1 |
| Число раундов | 8, 12, 16, 20, 24 |
| Стратегия | Sequential, Iterative |
| Top-K характеристик | 5, 10, 20 |
| Порог вероятности (log2) | -20, -30, -40 |
| Решатель | CryptoMiniSat |
| Таймаут на характеристику | 600с |

**Сравнительные метрики:**

| Метрика | Чистый SAT | Комбинированный |
|---------|-----------|----------------|
| Время до коллизии | T_sat | T_diff + T_encode + T_solve |
| Число конфликтов | N_sat | N_combined |
| Размер CNF (переменных) | V_sat | V_combined |
| Размер CNF (клозов) | C_sat | C_combined |
| Успешность | R_sat | R_combined |

## 8.5 Эксперимент 4: Масштабируемость

**Цель:** Определить максимальное число раундов, для которого комбинированный метод находит коллизию за фиксированное время.

**Методика:**
1. Фиксировать таймаут (1 час).
2. Запустить комбинированный метод для возрастающего числа раундов.
3. Определить границу практической применимости.

---

## 8.6 Эксперимент 5: Сравнение SAT-решателей

**Цель:** Выбрать оптимальный SAT-решатель для криптографических задач.

**Методика:**
1. Зафиксировать CNF-файл (одна и та же задача).
2. Запустить на нескольких решателях.
3. Сравнить время, конфликты, memory.

**Решатели:**
- CryptoMiniSat 5.x (с XOR-клозами и без)
- Glucose 4.x
- MiniSAT 2.2
- Cadical (последняя версия)

## 8.7 Управление экспериментами

```python
@dataclass
class ExperimentConfig:
    name: str
    hash_function: str
    num_rounds: list[int]
    solver: str
    timeout: int
    seed: int
    output_dir: str
    parameters: dict  # специфичные для эксперимента

class ExperimentManager:
    """Управление запуском и сохранением экспериментов."""

    def __init__(self, config: ExperimentConfig):
        self.config = config
        self.results = []

    def run(self):
        for rounds in self.config.num_rounds:
            result = self._run_single(rounds)
            self.results.append(result)
            self._save_intermediate(result)

    def _run_single(self, rounds: int) -> dict:
        start_time = time.time()
        # ... запуск эксперимента ...
        elapsed = time.time() - start_time
        return {
            "rounds": rounds,
            "time": elapsed,
            "result": ...,
            "stats": ...,
        }

    def save_results(self):
        """Сохранить результаты в JSON и CSV."""
        with open(f"{self.config.output_dir}/results.json", 'w') as f:
            json.dump(self.results, f, indent=2)
        df = pd.DataFrame(self.results)
        df.to_csv(f"{self.config.output_dir}/results.csv", index=False)
```

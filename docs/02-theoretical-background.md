# 2. Теоретические основы

## 2.1 Криптографические хэш-функции

### Определение

Криптографическая хэш-функция `H: {0,1}* → {0,1}^n` должна удовлетворять трём свойствам:

| Свойство | Формальное определение | Сложность (идеальная) |
|----------|----------------------|----------------------|
| **Стойкость к прообразу** | Дано `h`, найти `m`: `H(m) = h` | `O(2^n)` |
| **Стойкость ко второму прообразу** | Дано `m₁`, найти `m₂ ≠ m₁`: `H(m₁) = H(m₂)` | `O(2^n)` |
| **Стойкость к коллизиям** | Найти `m₁ ≠ m₂`: `H(m₁) = H(m₂)` | `O(2^{n/2})` (birthday bound) |

### Конструкция Меркла–Дамгорда

SHA-1 и SHA-256 используют конструкцию Меркла–Дамгорда:

```
M = m₁ ‖ m₂ ‖ ... ‖ mₖ  (разбиение на блоки)

H₀ = IV  (начальное значение)
Hᵢ = f(Hᵢ₋₁, mᵢ)  (функция сжатия)
H(M) = Hₖ
```

Функция сжатия `f` построена по схеме Дэвиса–Мейера:

```
f(H, m) = E_m(H) ⊕ H
```

где `E_m` — блочный шифр с ключом `m`.

Атака на функцию сжатия (нахождение коллизии для `f`) транслируется в атаку на всю хэш-функцию.

## 2.2 Дифференциальный криптоанализ

### Историческая справка

- **1990** — Бихам и Шамир предложили дифференциальный криптоанализ для блочных шифров (DES).
- **2004** — Ван и др. применили дифференциальный криптоанализ к MD5 и SHA-0, найдя практические коллизии.
- **2005** — Ван и др. нашли коллизии для полной SHA-1 с теоретической сложностью `2^{69}`.

### Основные понятия

**Дифференциал (разность):** Для двух входных значений `x` и `x'`, разность определяется как:

```
Δx = x ⊕ x'     (XOR-разность)
Δx = x - x'     (аддитивная разность, mod 2^w)
```

**Дифференциальная характеристика** — последовательность разностей через все раунды:

```
Δ₀ → Δ₁ → Δ₂ → ... → Δᵣ
```

где `Δᵢ` — разность на выходе i-го раунда.

**Вероятность характеристики:**

```
Pr[характеристика] = ∏ᵢ Pr[Δᵢ₋₁ → Δᵢ]
```

при предположении независимости раундов (Markov cipher assumption).

### Типы разностей для хэш-функций

| Тип разности | Операция | Формула |
|-------------|----------|---------|
| **XOR-разность** | `⊕` | `Δ = x ⊕ x'` |
| **Аддитивная разность** | `+` mod `2^w` | `Δ = (x - x') mod 2^w` |
| **Смешанная** | комбинация | зависит от раунда |

Для SHA-1/SHA-256 используются XOR-разности для булевых операций и аддитивные разности для модульного сложения.

### Распространение разностей через операции

**XOR:** `Δ(x ⊕ y) = Δx ⊕ Δy` — детерминированное, вероятность 1.

**AND:** `Δ(x ∧ y)` — зависит от конкретных значений. Для бита:
```
Pr[Δ(x ∧ y) = Δx ∧ y ⊕ x ∧ Δy ⊕ Δx ∧ Δy] = 1
```
но вероятность фиксированного выходного дифференциала зависит от Δx, Δy.

**Модульное сложение** `x + y mod 2^w`:
Распространение аддитивной разности:
```
(x + Δx) + (y + Δy) = (x + y) + (Δx + Δy) + carry_difference
```
Carry propagation вносит нелинейность — ключевой источник сложности анализа.

**Битовые сдвиги и ротации:**
```
Δ(x <<< r) = Δx <<< r   (ротация — линейна)
Δ(x >> r) = Δx >> r      (логический сдвиг — линеен)
```

### Автоматический поиск дифференциальных характеристик

Методы поиска:

1. **Перебор с ветвлением и отсечением (Branch and Bound):**
   - Строим характеристику раунд за раундом.
   - Отсекаем ветви с вероятностью ниже порога.
   - Сложность: экспоненциальная, но управляемая для малого числа раундов.

2. **MILP (Mixed Integer Linear Programming):**
   - Кодирование дифференциальных ограничений как система линейных неравенств.
   - Минимизация числа активных S-блоков.
   - Применяется для AES-подобных конструкций; для ARX-конструкций (SHA) менее эффективен.

3. **SAT/SMT-based поиск:**
   - Кодирование ограничений на дифференциалы как SAT/SMT задача.
   - Позволяет точно учитывать все нелинейные операции.
   - Именно этот подход ложится в основу комбинированного метода.

## 2.3 SAT-based криптоанализ

### Проблема булевой выполнимости (SAT)

**Определение:** Дана булева формула `φ(x₁, ..., xₙ)` в конъюнктивной нормальной форме (CNF). Найти назначение переменных, при котором `φ = true`, или доказать, что такого назначения не существует.

**CNF-формат:**
```
φ = C₁ ∧ C₂ ∧ ... ∧ Cₘ
```
где каждый клоз `Cⱼ` — дизъюнкция литералов: `Cⱼ = (l₁ ∨ l₂ ∨ ... ∨ lₖ)`.

**SAT является NP-полной задачей** (теорема Кука, 1971), но современные решатели эффективно обрабатывают промышленные экземпляры с миллионами переменных.

### Алгоритм DPLL и CDCL

Современные SAT-решатели основаны на алгоритме **CDCL** (Conflict-Driven Clause Learning):

```
1. Unit Propagation (BCP)
   - Если клоз содержит одну не-назначенную переменную — присвоить значение.
2. Decision
   - Выбрать не-назначенную переменную, присвоить значение.
3. Conflict Analysis
   - При конфликте: найти UIP (Unique Implication Point).
   - Построить learnt clause.
4. Backjump
   - Откатиться до нужного уровня решения.
5. Restart
   - Периодические перезапуски для выхода из тупиков.
```

### Кодирование криптографических задач в SAT

Идея: представить раундовые операции хэш-функции как систему булевых ограничений.

**Переменные:** каждый бит внутреннего состояния, сообщения и промежуточных значений — отдельная булева переменная.

**Ограничения:** каждая операция (XOR, AND, модульное сложение) кодируется как набор клозов.

**Дополнительные ограничения:**
- Фиксация входной/выходной разности.
- Фиксация известных битов.
- Ограничения на дифференциальные условия.

### Ключевые работы по SAT-криптоанализу

| Год | Авторы | Вклад |
|-----|--------|-------|
| 2000 | Massacci, Marraro | Первое применение SAT к DES |
| 2006 | Mironov, Zhang | SAT-based атаки на MD4, MD5 |
| 2007 | De, Kuhn, Bhatt | SAT-кодирование SHA-1 |
| 2011 | Nejati et al. | CryptoMiniSat для криптоанализа |
| 2012 | Homsirikamol et al. | SAT для SHA-256 (уменьшенные раунды) |
| 2020 | Nejati et al. | Комбинирование CDCL с algebraic reasoning |

## 2.4 Комбинированный подход

### Мотивация

Чистый дифференциальный анализ находит характеристики, но не конкретные пары сообщений. Чистый SAT-подход не использует структурную информацию и работает «вслепую».

**Идея комбинирования:**

```
Дифференциальный анализ    →  Сужение пространства поиска
        ↓
SAT-решатель              →  Нахождение конкретных решений
        ↓
Верификация               →  Подтверждение коллизий
```

### Формализация

Пусть `D` — множество дифференциальных характеристик с вероятностью ≥ `p_min`.

Для каждой характеристики `d ∈ D`:
1. Закодировать хэш-функцию как SAT-задачу `φ_hash`.
2. Добавить дифференциальные ограничения `φ_diff(d)`.
3. Решить `φ_hash ∧ φ_diff(d)`.
4. Если SAT — извлечь пару сообщений `(m, m')`.
5. Верифицировать: `H(m) = H(m')`.

### Преимущества комбинирования

| Аспект | Чистый дифф. | Чистый SAT | Комбинированный |
|--------|-------------|-----------|----------------|
| Размер SAT-задачи | — | Огромный | Сокращённый |
| Использование структуры | Да | Нет | Да |
| Гарантия нахождения | Нет | Да (при SAT) | Да (при SAT) |
| Масштабируемость | Средняя | Низкая | Высокая |

## 2.5 Смежные исследования

### ARX-конструкции и их анализ

SHA-1 и SHA-256 принадлежат к классу **ARX** (Addition-Rotation-XOR). Специфика анализа:

- Модульное сложение — единственный нелинейный элемент.
- Ротации и XOR — линейны над `GF(2)`.
- Сложность анализа определяется взаимодействием `+` и `⊕`.

### Связь с SMT (Satisfiability Modulo Theories)

SMT расширяет SAT теориями:
- Теория битовых векторов (`QF_BV`) — точное моделирование слов фиксированной разрядности.
- Потенциально более компактное кодирование модульного сложения.
- Инструменты: Z3, Boolector, CVC5.

В рамках ВКР основной фокус — на чистом SAT с возможным расширением на SMT в будущей работе.

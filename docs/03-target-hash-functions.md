# 3. Целевые хэш-функции

## 3.1 Обоснование выбора

Для экспериментальной оценки комбинированного метода выбраны **SHA-256** и **SHA-1** (уменьшенные варианты):

| Критерий | SHA-256 | SHA-1 |
|----------|---------|-------|
| Практическая значимость | Используется повсеместно (TLS, Bitcoin, Git) | Устаревший, но хорошо изученный |
| Наличие известных атак | До 31 раунда (дифференциальные) | Полная коллизия (80 раундов, 2017) |
| Сложность одного раунда | Высокая (8 рабочих слов, 6 операций) | Средняя (5 рабочих слов, 3 операции) |
| Число раундов (полная) | 64 | 80 |
| Размер слова | 32 бит | 32 бит |
| Размер хэша | 256 бит | 160 бит |

Уменьшенные варианты (reduced-round) позволяют получить результаты за разумное время, при этом сохраняя структурную сложность полных версий.

## 3.2 SHA-256

### Общая структура

SHA-256 обрабатывает 512-битные блоки сообщения. Внутреннее состояние — 8 слов по 32 бита: `a, b, c, d, e, f, g, h`.

### Раундовая функция

Один раунд SHA-256 (i-й раунд, `i = 0..63`):

```
T₁ = h + Σ₁(e) + Ch(e, f, g) + Kᵢ + Wᵢ
T₂ = Σ₀(a) + Maj(a, b, c)

h = g
g = f
f = e
e = d + T₁
d = c
c = b
b = a
a = T₁ + T₂
```

### Операции

```
Ch(x, y, z)  = (x ∧ y) ⊕ (¬x ∧ z)
Maj(x, y, z) = (x ∧ y) ⊕ (x ∧ z) ⊕ (y ∧ z)
Σ₀(x) = ROTR²(x) ⊕ ROTR¹³(x) ⊕ ROTR²²(x)
Σ₁(x) = ROTR⁶(x) ⊕ ROTR¹¹(x) ⊕ ROTR²⁵(x)
```

### Расписание сообщения (Message Schedule)

```
Wᵢ = Mᵢ                                      для i = 0..15
Wᵢ = σ₁(Wᵢ₋₂) + Wᵢ₋₇ + σ₀(Wᵢ₋₁₅) + Wᵢ₋₁₆  для i = 16..63

σ₀(x) = ROTR⁷(x) ⊕ ROTR¹⁸(x) ⊕ SHR³(x)
σ₁(x) = ROTR¹⁷(x) ⊕ ROTR¹⁹(x) ⊕ SHR¹⁰(x)
```

### Дифференциальные свойства SHA-256

**Нелинейные компоненты:**
- `Ch(x, y, z)` — условный выбор, нелинеен по всем аргументам.
- `Maj(x, y, z)` — мажоритарная функция, нелинейна.
- Модульное сложение `+` — основной источник нелинейности.

**Линейные компоненты:**
- Ротации `ROTR` и сдвиги `SHR` — линейны над `GF(2)`.
- XOR — линеен.

**Таблица дифференциальных свойств операций:**

| Операция | Тип | Вероятность дифференциала |
|----------|-----|-------------------------|
| `x ⊕ y` | Линейная | 1 |
| `ROTR(x, r)` | Линейная | 1 |
| `Ch(x, y, z)` | Нелинейная | зависит от позиции активных бит |
| `Maj(x, y, z)` | Нелинейная | зависит от позиции активных бит |
| `x + y mod 2³²` | Нелинейная | `≈ 2^{-k}` где `k` — число carry-пар |

### Известные результаты криптоанализа SHA-256

| Раунды | Тип атаки | Авторы | Год | Сложность |
|--------|-----------|--------|-----|-----------|
| 18 | Коллизия | Indesteege et al. | 2009 | Практическая |
| 24 | Коллизия (free-start) | Mendel et al. | 2011 | Практическая |
| 28 | Pseudo-коллизия | Eichlseder et al. | 2020 | 2^{50} |
| 31 | Дифференциальная характеристика | Leurent, Peyrin | 2020 | Теоретическая |

### Реализация для проекта

Модуль `src/hash_functions/sha256.py` должен содержать:

```python
class SHA256Reduced:
    """Уменьшенный вариант SHA-256 с настраиваемым числом раундов."""

    def __init__(self, num_rounds: int = 64):
        self.num_rounds = num_rounds
        self.K = [...]  # Константы раундов
        self.H0 = [...]  # Начальное состояние (IV)

    def compress(self, state: list[int], block: list[int]) -> list[int]:
        """Функция сжатия: один блок."""
        W = self._message_schedule(block)
        a, b, c, d, e, f, g, h = state

        for i in range(self.num_rounds):
            T1 = (h + self._sigma1(e) + self._ch(e, f, g)
                   + self.K[i] + W[i]) & 0xFFFFFFFF
            T2 = (self._sigma0(a) + self._maj(a, b, c)) & 0xFFFFFFFF
            h, g, f, e = g, f, e, (d + T1) & 0xFFFFFFFF
            d, c, b, a = c, b, a, (T1 + T2) & 0xFFFFFFFF

        return [(s + v) & 0xFFFFFFFF for s, v in zip(state, [a,b,c,d,e,f,g,h])]

    def _ch(self, x, y, z):
        return (x & y) ^ (~x & z)

    def _maj(self, x, y, z):
        return (x & y) ^ (x & z) ^ (y & z)

    def _sigma0(self, x):
        return self._rotr(x,2) ^ self._rotr(x,13) ^ self._rotr(x,22)

    def _sigma1(self, x):
        return self._rotr(x,6) ^ self._rotr(x,11) ^ self._rotr(x,25)

    def _rotr(self, x, n):
        return ((x >> n) | (x << (32 - n))) & 0xFFFFFFFF
```

## 3.3 SHA-1

### Общая структура

SHA-1 обрабатывает 512-битные блоки. Внутреннее состояние — 5 слов по 32 бита: `a, b, c, d, e`.

### Раундовая функция

```
T = (a <<< 5) + fᵢ(b, c, d) + e + Kᵢ + Wᵢ
e = d
d = c
c = b <<< 30
b = a
a = T
```

### Раундовые булевы функции

| Раунды | Функция `fᵢ` | Название |
|--------|-------------|----------|
| 0–19 | `(b ∧ c) ⊕ (¬b ∧ d)` | Ch |
| 20–39 | `b ⊕ c ⊕ d` | Parity |
| 40–59 | `(b ∧ c) ⊕ (b ∧ d) ⊕ (c ∧ d)` | Maj |
| 60–79 | `b ⊕ c ⊕ d` | Parity |

### Message Schedule

```
Wᵢ = Mᵢ                                    для i = 0..15
Wᵢ = (Wᵢ₋₃ ⊕ Wᵢ₋₈ ⊕ Wᵢ₋₁₄ ⊕ Wᵢ₋₁₆) <<< 1  для i = 16..79
```

### Известные атаки на SHA-1

| Год | Авторы | Результат |
|-----|--------|-----------|
| 2005 | Wang et al. | Теоретическая коллизия, 2^{69} |
| 2017 | Stevens et al. (SHAttered) | Первая практическая коллизия |
| 2020 | Leurent, Peyrin | Chosen-prefix коллизия, 2^{63.4} |

SHA-1 официально deprecated и представляет интерес как объект исследования с хорошо изученной дифференциальной структурой.

### Реализация для проекта

```python
class SHA1Reduced:
    """Уменьшенный вариант SHA-1 с настраиваемым числом раундов."""

    def __init__(self, num_rounds: int = 80):
        self.num_rounds = num_rounds
        self.K = [0x5A827999, 0x6ED9EBA1, 0x8F1BBCDC, 0xCA62C1D6]
        self.H0 = [0x67452301, 0xEFCDAB89, 0x98BADCFE, 0x10325476, 0xC3D2E1F0]

    def compress(self, state: list[int], block: list[int]) -> list[int]:
        W = self._message_schedule(block)
        a, b, c, d, e = state

        for i in range(self.num_rounds):
            f, k = self._round_function(i, b, c, d)
            T = (self._rotl(a,5) + f + e + k + W[i]) & 0xFFFFFFFF
            e, d, c, b, a = d, c, self._rotl(b,30), a, T

        return [(s + v) & 0xFFFFFFFF for s, v in zip(state, [a,b,c,d,e])]
```

## 3.4 Опциональные цели

### MD5

- 128-битный хэш, 64 раунда.
- Полностью сломан (практические коллизии с 2004 года).
- Полезен для тестирования и валидации фреймворка.

### SHA-3 / Keccak (уменьшенные варианты)

- Принципиально другая конструкция (sponge).
- Требует иного подхода к дифференциальному анализу.
- Рассматривается как перспективное расширение, не входит в основной объём ВКР.

## 3.5 Параметры экспериментов

| Хэш-функция | Раунды для экспериментов | Обоснование |
|-------------|-------------------------|-------------|
| SHA-256 | 8, 12, 16, 20, 24 | Постепенное увеличение сложности |

Для каждого варианта хэш-функции необходимо:
1. Верифицировать реализацию на тестовых векторах (NIST).
2. Построить побитовую модель раундовой функции.
3. Определить дифференциальные свойства каждой операции.
4. Сформировать SAT-кодирование.
